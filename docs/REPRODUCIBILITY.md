# Reproducibility Policy

Aethermor guarantees deterministic, reproducible outputs across releases.

---

## Fixed Seed Policy

All stochastic components use fixed seeds by default:

- **NumPy**: `np.random.seed(42)` in all benchmark and simulation scripts
- **Python random**: `random.seed(42)` where used
- **Environment override**: Set `AETHERMOR_SEED` to override the default seed

The physics models (thermal solver, energy models, material properties) are
fully deterministic — no randomness involved.

## Numerical Tolerance Policy

| Component | Tolerance | Rationale |
|-----------|-----------|-----------|
| Fundamental constants | Exact (0.0%) | CODATA 2018 definitions |
| Material properties | ≤ 10% vs CRC Handbook | Published value ranges across sources |
| Analytical solutions | Exact (0.0%) | Closed-form implementations |
| 3D Fourier energy conservation | < 5.0% | Explicit time-stepping with CFL stability |
| Benchmark regression | ≤ 1.0% | Gold output comparison |
| Cross-reference temperature | ≤ 0.1 K | Round-trip and self-consistency |

## Gold Outputs

The `benchmarks/gold_outputs/` directory contains frozen reference outputs
from each production benchmark case. These are generated by
`scripts/freeze_gold_outputs.py` and checked on every release.

| File | Description |
|------|-------------|
| `production_suite_v1.0.0.json` | Reference outputs for the production benchmark suite |

### Regenerating Gold Outputs

```bash
python scripts/freeze_gold_outputs.py
```

This overwrites the gold outputs with current model outputs. Only do this
when intentionally changing model behavior, and document the reason in the
release notes.

## Release Regression Report

Each release includes a benchmark regression report at:

```
reports/release_v<version>_benchmark_report.md
```

This report documents:

- Whether accuracy improved or regressed vs the previous release
- Any gold output changes and their justification
- Whether the release is safe for routine use

## Reproducibility Verification

```bash
# Verify deterministic outputs
python scripts/reproduce_strict.sh   # Linux/macOS
python scripts/reproduce_strict.ps1  # Windows

# Verify gold outputs match
python scripts/freeze_gold_outputs.py --verify-only
```

## What Triggers a Gold Output Update

1. Intentional model improvement (e.g., better material data)
2. Bug fix that changes numerical outputs
3. New benchmark cases added

All gold output changes are documented in [CHANGELOG.md](../CHANGELOG.md)
with the reason for the change.
