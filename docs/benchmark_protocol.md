# Benchmark Protocol

## Purpose

This document defines the production benchmark protocol for Aethermor.
Every release must pass this benchmark suite before shipping.

---

## Production Benchmark Suite

### Location

```
benchmarks/production_suite/
  cases.csv                    # 20 benchmark cases (hardware + synthetic)
  run_production_suite.py      # Runner with gate checks
```

### Case Composition

| Category | Cases | Description |
|----------|-------|-------------|
| Real hardware (measured θ_jc) | 3 | A100, i9-13900K, Ryzen 7950X |
| Real hardware (published specs) | 12 | H100, MI300X, EPYC, Xeon, M1, M2, Snapdragon, etc. |
| Heterogeneous SoC | 1 | Synthetic multi-block SoC |
| Paradigm crossover | 1 | CMOS vs adiabatic synthetic |
| Cooling tradeoff | 1 | Variable cooling synthetic |
| Material comparison | 2 | Diamond spreader, SiC vs Si |

**Total: 20 cases across 4 market segments + 5 synthetic engineering scenarios.**

---

## Release-Gating Thresholds

| Gate | Threshold | Measured By |
|------|-----------|-------------|
| Material median error | ≤ 1.0% | Material property vs CRC Handbook |
| Material worst-case error | ≤ 10.0% | Worst single property |
| Energy conservation | ≤ 5.0% | 3D Fourier solver |
| Gold regression | ≤ 1.0% per case | Current vs frozen gold outputs |
| All cases valid | 100% | Every case produces finite, positive Tj |

## Suite Classification

### Physics Consistency Suites

These verify that the physics models are internally consistent and
agree with fundamental references.

| Suite | Checks | What It Verifies |
|-------|--------|------------------|
| `validation/validate_all.py` | 133 | Constants, materials, energy models, solver, roadmap |
| `benchmarks/literature_validation.py` | 20 | CODATA, CRC, ITRS/IRDS, Incropera textbook |
| `benchmarks/material_cross_validation.py` | 93 | 9 materials vs 3+ independent sources each |

### Engineering Decision Reliability Suites

These verify that Aethermor produces correct answers for real engineering
questions with known outcomes.

| Suite | Checks | What It Verifies |
|-------|--------|------------------|
| `benchmarks/chip_thermal_database.py` | 82 | 12 real chips, correct temperature ranges |
| `benchmarks/real_world_validation.py` | 33 | 4 chip designs, full thermal analysis |
| `benchmarks/experimental_validation.py` | 18 | JEDEC θ_jc, IR thermal, HotSpot benchmark |
| `benchmarks/case_study_datacenter.py` | 13 | Datacenter cooling decision |
| `benchmarks/case_study_mobile_soc.py` | 10 | Mobile SoC thermal envelope |
| `benchmarks/production_suite/run_production_suite.py` | 20 | Release gate cases |

### Informational Suites

These provide additional context but are not release gates.

| Suite | What It Shows |
|-------|---------------|
| `benchmarks/hotspot_comparison.py` | Comparison with HotSpot capabilities |
| `benchmarks/case_study_cooling_decision.py` | Cooling vs substrate tradeoff demo |
| `benchmarks/case_study_substrate_selection.py` | Material ranking demo |
| `benchmarks/case_study_soc_bottleneck.py` | SoC reallocation demo |

---

## Running the Suite

```bash
# Full production gate check
python benchmarks/production_suite/run_production_suite.py

# Full validation (all suites, 680+ checks)
python run_all_validations.py
```

---

## Gold Outputs

Frozen reference outputs are stored in `benchmarks/gold_outputs/`.
These are generated by `scripts/freeze_gold_outputs.py` and checked
on every release. See [docs/REPRODUCIBILITY.md](../docs/REPRODUCIBILITY.md)
for the full reproducibility policy.

---

## Benchmark Report

Each release includes a benchmark report at:

```
reports/release_v<version>_benchmark_report.md
```

This report answers:
1. Did accuracy improve or regress?
2. Is this release safe for routine use?
3. What changed and why?
